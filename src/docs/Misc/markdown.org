* Phrase Emphasis

#+end_src

* Lists
If you put blank lines between items, you’ll get <p> tags for the list
item text. You can create multi-paragraph list items by indenting the
paragraphs by *4 spaces* or *1 tab*:

#+begin_src sh
  *   A list item.
  
      With multiple paragraphs.
  
      *   Another item in the list.
#+end_src

Output:

#+begin_src html
<ul>
<li><p>A list item.</p>
<p>With multiple paragraphs.</p></li>
<li><p>Another item in the list.</p></li>
</ul>
#+end_src

To put a code block within a list item, the code block needs to be
indented twice — 8 spaces or two tabs:

#+begin_src sh
 *   A list item with a code block:

        <code goes here>
#+end_src

In other words, a number-period-space sequence at the beginning of a line. To avoid this, you can backslash-escape the period:

1986\. What a great season.

* Links

** links: inline
#+begin_src sh
This is an [example link](http://example.com/).

#+end_src

** reference
#+begin_src sh
I get 10 times more traffic from [Google][1] than from
[Yahoo][2] or [MSN][3].

[1]: http://google.com/        "Google"
[2]: http://search.yahoo.com/  "Yahoo Search"
[3]: http://search.msn.com/    "MSN Search"
#+end_src

* Images
#+begin_src sh
Inline (titles are optional):

![alt text](/path/to/img.jpg "Title")

Reference-style:

![alt text][id]

[id]: /path/to/img.jpg "Title"
#+end_src

* Code
To specify an entire block of pre-formatted code, indent every line of
the block by 4 spaces or 1 tab. Just like with code spans, &, <, and >
characters will be escaped automatically.

or example, given this input:
#+begin_src sh
This is a normal paragraph:

    This is a code block.
#+end_src

Markdown will generate:
#+begin_src html
<p>This is a normal paragraph:</p>

<pre><code>This is a code block.
</code></pre>
#+end_src

To include a literal backtick character within a code span, you can use multiple backticks as the opening and closing delimiters:
#+begin_src sh
``There is a literal backtick (`) here.``
#+end_src

which will produce this:
#+begin_src html
<p><code>There is a literal backtick (`) here.</code></p>
#+end_src

* Misc
Markdown provides backslash escapes for the following characters:

#+begin_src sh
  \   backslash
  `   backtick
  *   asterisk
  _   underscore
  {}  curly braces
  []  square brackets
  ()  parentheses
  #   hash mark
  +   plus sign
  -   minus sign (hyphen)
  .   dot
  !   exclamation mark
#+end_src
